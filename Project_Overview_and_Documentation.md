# Project Overview and Documentation

## Project Title
CMSC 495 Bookkeeping Project

## Project Overview
The CMSC 495 Bookkeeping Project is a web-based bookkeeping system created as a university capstone project. Its purpose is to give small business owners a simple way to manage financial records, business documents, customers, vendors, and account activity from a single application.

The project was developed by a six-person team organized into front-end and back-end groups. The front-end effort focused on the user interface, styling, and user interaction, while the back-end effort handled database structure and RESTful integration between the interface and the data layer.

The finished system is designed to help users maintain organized financial records without relying on manual spreadsheets or disconnected tools. It provides account-based views of transactions and documents, along with a dashboard that makes bookkeeping information easier to review and manage.

## Problem Statement
Small businesses often need a lightweight bookkeeping solution that is easier to use than full accounting suites but more structured than manual recordkeeping. This project addresses that gap by offering a centralized application for storing and reviewing bookkeeping data in a clear, accessible format.

## Project Goals
- Build a simple bookkeeping platform for small business use.
- Support user account creation and secure login.
- Allow users to manage customers, vendors, financial accounts, and bookkeeping documents.
- Present transaction details in a clear double-entry ledger style.
- Provide a dashboard view for reviewing existing records quickly.
- Support practical administrative actions such as printing records and changing passwords.

## Core Features
- User authentication with account-based access
- Customer management
- Vendor management
- Financial account management
- Document entry and document tracking
- Transaction listing and ledger-style financial views
- Dashboard tables for reviewing saved records
- Printing support for recordkeeping
- Password update functionality

## Technology Summary
Based on the available project materials and repository summary, the system uses:

- JavaScript for core front-end behavior
- PHP for server-side processing
- CSS for presentation and layout
- HTML for page structure
- REST-style communication between the interface and back-end components
- A database layer managed by the back-end team

## Intended Users
The main users are small business owners and other users who need a basic bookkeeping tool for:

- Tracking business accounts
- Managing customers and vendors
- Recording documents and transactions
- Reviewing ledger activity
- Maintaining printable financial records

## High-Level Architecture
The application follows a standard web application structure:

1. A front-end interface accepts and validates user input.
2. The interface communicates with back-end RESTful endpoints.
3. The back-end processes requests and updates bookkeeping records in the database.
4. Results are returned to the dashboard and record views for user review.

This separation of concerns supports maintainability and allows front-end and back-end work to progress in parallel.
